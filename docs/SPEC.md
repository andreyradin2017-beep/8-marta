# Спецификация проекта: Поздравительная страница "8 Марта"

## 1. Общее описание

Проект представляет собой одностраничный веб-сайт (Landing Page), разработанный для поздравления с наступающим праздником 8 Марта от лица "Маркетинговой группы Текарт".
Документ содержит фоновое видео, кроссбраузерные параллакс-эффекты для декоративных элементов, изящную типографику и возможность управления звуком видеоролика.

## 2. Технологический стек

Проект реализован без использования тяжелых фреймворков для обеспечения максимальной скорости загрузки и простоты поддержки:

- **HTML5:** Семантическая разметка (использование тегов `<main>`, `<section>`, `<footer>`, `<header>`).
- **CSS3:** Стилизация, использование CSS-переменных, Flexbox/Grid для адаптивной верстки, медиазапросы, CSS-анимации (`keyframes` и `transitions`).
- **JavaScript (Vanilla JS):** Логика управления звуком видео и реализация параллакс-эффектов при движении мыши (Mousemove Parallax).
- **Шрифты:** Интеграция шрифта `Onest` (Google Fonts) и локального шрифта для заголовков (если применимо).

## 4. Описание основных компонентов

### 4.1 HTML Разметка (`index.html`)

- **`<head>`:** Содержит необходимые мета-теги для SEO и мобильного отображения (`viewport`), подключение шрифта Onest, подключение `css/style.css`.
- **Фоновое видео (`.background-video`):** Используется видео-тег `<video>` с ключевыми параметрами `autoplay`, `loop`, `muted`, `playsinline` (для автозапуска на iOS).
- **Контентная часть (`<main class="main-content">`):**
  - Логотип компании (`.header__logo`).
  - Кнопка переключения звука (`#sound-toggle`).
  - Декоративные элементы (`.parallax-layer`) для создания эффекта многослойности. В разметке заданы Data-атрибуты `data-speed` для расчета скорости параллакса в JS.
  - Основной поздравительный текст (`.greeting`).
- **Подвал (`<footer class="footer">`):** Содержит текст копирайта и внешнюю ссылку на сайт "Текарт".

### 4.2 Стилизация (`css/style.css`)

- **Адаптивность:** Элементы страницы адаптируются под различные размеры экранов от смартфонов до широкоформатных мониторов. Используются функции `clamp()`, `vw/vh` (Viewport units) и блок `@media` запросов.
- **Z-index архитектура:** Строгая иерархия слоев:
  - Фоновое видео: `z-index: 0`
  - Затемняющий слой (overlay): `z-index: 10`
  - Декоративные элементы: `z-index: 20`
  - Контент и текст: `z-index: 50`
  - Элементы управления (UI): `z-index: 100`
- **Визуальные эффекты:**
  - Затенение фона (overlay mask) для обеспечения высокой читаемости светлого текста поверх динамичного видеофона.
  - Плавное появление (Fade-in animations) элементов при начальной загрузке страницы.
  - Плавная смена состояний (hover/focus эффекты на кнопках и активных ссылках).

### 4.3 Скрипты (`js/main.js`)

- **Взаимодействие со звуком (Mute / Unmute):**
  1. Отслеживание клика по кнопке `#sound-toggle`.
  2. Переключение свойства `muted` у тега `<video id="bg-video">`.
  3. Визуальная смена состояния кнопки или иконки в зависимости от режима (звук включен/выключен).
- **Эффект параллакса (Mousemove Parallax):**
  1. Отслеживание события `mousemove` на объекте `window` или `document`.
  2. Расчет позиции курсора относительно центра экрана.
  3. Смещение элементов с классом `.js-parallax` посредством CSS `transform: translate()` пропорционально движению курсора мыши и заданному множителю `data-speed`.

## 5. Требования к ресурсам

- **Видео:** `video.mp4` – рекомендуется использовать компрессированное видео (кодек H.264) без лишних и пустых аудиотреков, с битрейтом, оптимальным для Web (рекомендуется 2-5 Mbps для 1080p).
- **Изображения:** Все иконки и логотипы компании должны быть выполнены в векторном формате (SVG) для идеальной четкости и рендеринга на Retina-дисплеях, а также обеспечения малого размера файла.

## 6. Браузерная поддержка

Проект должен корректно отображаться и функционировать в современных браузерах:

- Google Chrome (последние версии)
- Safari (macOS / iOS 14+)
- Mozilla Firefox (последние версии)
- Microsoft Edge (последние версии)
- Мобильные браузеры (Chrome for Android, Safari for iOS)

## 7. Развертывание

Так как клиентская часть проекта состоит исключительно из статических файлов (Static Site), он может быть легко развернут на любой хостинг-платформе (например, Vercel, Netlify, GitHub Pages, Amazon S3) или загружен напрямую на выделенный web-сервер (Nginx / Apache) без конфигурации бэкенда.
